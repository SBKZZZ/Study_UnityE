1. 커밋 메시지 구조
<타입>: <제목>
<본문>
<푸터>

2. 커밋 메시지 타입
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 변경
style: 코드 포맷팅, 세미콜론 누락 등
refactor: 코드 리팩토링 (기능 변경x)
test: 테스트 추가 또는 수정
chore: 빌드/보조 도구 수정, 라이브러리 추가 등
perf: 성능 향상 관련 변경
build: 빌드 관련 파일 변경
ci: CI 설정 관련 변경

3. 제목 작성 규칙
50자 이내로 간결하게
첫 글자는 대문자로
마침표 사용하지 않음
명령문 형태로 작성
예시: Feat: Add user login feature Fix: Resolve image rendering bug

4. 본문 작성 규칙 (선택 사항)
제목과 본문 사이에 한 줄 띄움
각 행은 72자 이내로 작성
"무엇을" "왜" 변경했는지 중심으로 설명
"어떻게" 변경했는지는 코드에 드러나므로 작성하지 않아도 됨

5. 푸터 작성 규칙 (선택 사항)
이슈 번호나 브레이킹 체인지(Breaking change) 등을 명시할 때 사용
예시 : BREAKING CHANGE: Refactored user model API Closes #123
